## рҹ”җ Security Features

### Network Security
- вң… **VPC isolation** вҖ” 3-tier subnet architecture (public / private / database)
- вң… **No public RDS access** вҖ” database subnets have no internet route
- вң… **NAT Gateways** вҖ” worker nodes egress-only, never directly exposed
- вң… **Security groups** вҖ” MySQL port 3306 restricted to VPC CIDR only
- вң… **EKS private endpoint** вҖ” control plane accessible from within VPC

### Encryption
- вң… **RDS encryption at rest** вҖ” AES-256 via dedicated KMS key
- вң… **EKS secrets encryption** вҖ” Kubernetes secrets encrypted with KMS
- вң… **ECR image encryption** вҖ” KMS encrypted container images
- вң… **TLS in transit** вҖ” Let's Encrypt certificates via cert-manager
- вң… **S3 state encryption** вҖ” SSE-AES256 on Terraform state bucket

### IAM & Access Control
- вң… **Least privilege IAM** вҖ” separate roles for EKS cluster and node groups
- вң… **No hardcoded credentials** вҖ” all secrets via GitHub Actions secrets
- вң… **SSM access** вҖ” node access via SSM, no SSH keys needed
- вң… **Production environment** вҖ” manual approval required before apply/destroy

### Data Protection
- вң… **RDS Multi-AZ** вҖ” synchronous replication across 2 AZs
- вң… **Automated backups** вҖ” 7-day retention with point-in-time recovery
- вң… **Deletion protection** вҖ” RDS cannot be accidentally deleted
- вң… **Final snapshot** вҖ” snapshot taken before any RDS deletion
- вң… **ECR immutable tags** вҖ” container images cannot be overwritten

## рҹҡҖ Deployment Guide

### Prerequisites

```bash
# Required tools
terraform >= 1.6.0
aws-cli >= 2.x
kubectl >= 1.28
helm >= 3.x
docker >= 24.x
git
```

### Step 1 вҖ” Bootstrap Remote State (run once)

```bash
cd bootstrap/
terraform init
terraform apply
# Copy the output bucket name into backend.tf
```

### Step 2 вҖ” Configure GitHub Actions Secrets

Add these secrets to your GitHub repository:

| Secret | Description |
|---|---|
| `AWS_ACCESS_KEY_ID` | IAM user access key |
| `AWS_SECRET_ACCESS_KEY` | IAM user secret key |
| `AWS_REGION` | Target region (us-east-1) |
| `TF_VAR_DB_USERNAME` | RDS master username |
| `TF_VAR_DB_PASSWORD` | RDS master password (min 16 chars) |
| `TF_VAR_DB_NAME` | Database name |
| `TF_VAR_ZONE_NAME` | Your domain (e.g. example.com) |
| `TF_VAR_DOMAIN_NAME` | App domain (e.g. example.com) |
| `TF_VAR_EMAIL` | Email for TLS certificates |

### Step 3 вҖ” Deploy Infrastructure

```bash
# Clone the repo
git clone https://github.com/samklin92/terraform-eks-banking-app.git
cd terraform-eks-banking-app

# Initialise
terraform init

# Preview changes
terraform plan

# Deploy (triggers GitHub Actions with manual approval)
git push origin main
```

### Step 4 вҖ” Configure kubectl

```bash
aws eks update-kubeconfig \
  --name production-eks-cluster \
  --region us-east-1
kubectl get nodes
```

### Step 5 вҖ” Apply ClusterIssuer for TLS

```bash
kubectl apply -f ClusterIssuer.yaml
kubectl get clusterissuer
```

### Step 6 вҖ” Deploy Application via Argo CD

```bash
# Access Argo CD
kubectl port-forward svc/argocd-server -n argocd 8080:80

# Get admin password
kubectl -n argocd get secret argocd-initial-admin-secret \
  -o jsonpath='{.data.password}' | base64 -d

# Deploy application
kubectl apply -f argocd-banking-app.yaml
```

### Step 7 вҖ” Update DNS

Get nameservers from Terraform output:
```bash
terraform output route53_name_servers
```
Point your domain's nameservers to the 4 Route 53 nameservers.
